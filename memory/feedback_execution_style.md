브레인스토밍·설계 단계를 최소화하고 빠른 실행을 선호한다.

**Why:** 이번 세션에서 디자인 문서 작성·스펙 리뷰 단계를 건너뛰고 바로 배포까지 진행했으며, 사용자가 이에 이의를 제기하지 않았음.

**How to apply:**
- 선택지 질문은 브라우저 비주얼로 빠르게 제시하고 단답 응답을 기다린다.
- 설계 문서(spec), 구현 계획(writing-plans) 단계는 사용자가 명시적으로 요청할 때만 진행.
- 결정이 나면 즉시 코드 작성·커밋·배포까지 이어서 진행.
- 응답은 짧고 결과 중심으로. 중간 확인 질문 최소화.
